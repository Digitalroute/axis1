今日は何をしたいですか？

  • ローカルインストールの設定の検証する
    うまく機能しない場合は「Axisの検証」を参照
  • 配備済のWebサービスのリストを見る
  • ローカルのエンドポイントを呼び出す - そのリストは呼び出し元のhttpヘッダです (もしくは WSDLを参照して下さい).
  • Apache-Axisホームページを訪問する
  • Axisを管理する
    [安全性の理由からデフォルトでは利用不可]
  • SOAPモニタ
    [安全性の理由からデフォルトでは利用不可]
上記のデフォルトで利用できない機能を有効にするには、webアプリケーション内の WEB-INF/web.xmlファイルの該当する宣言のコメントを外し、アプリケーションを再起動して下さい。

Axisの検証

"happyaxis"検証ページが状態表示のページではなく例外を表示する場合、 原因としてはクラスパス内に複数のXMLパーサを指定していることが考えられます。 関係のないパーサをクラスパスから取り除いてみて下さい。

Axisを動作させる上で問題を抱えている場合は、 Axis Wiki に意見を聞き、その後Axisユーザメーリングリストに投稿してみて下さい。
